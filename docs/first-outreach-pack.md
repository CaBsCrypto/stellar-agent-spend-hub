# First Outreach Pack

## Short DM

Hey, I am testing a Stellar-first agentic payments demo. The idea is simple: an AI agent prepares a USDC payment for a digital service, but the user keeps control and approves every settlement.

Could you open this and tell me what is clear or confusing after two minutes?

https://agente-pagos-stellar.vercel.app

The best feedback is: what you think it does, whether you would trust it, and what feels missing before using it with real providers.

## Stellar / SCF angle

We are building a Stellar-first Agent Spend Hub: MCP/API providers can expose paid resources, agents can prepare payments, and users approve under policy with privacy-safe receipts.

Current proof points:
- Public deployed demo.
- Stellar testnet focus.
- MPP and Contract Account architecture.
- Human approval required.
- Submit gates closed outside supervised tests.

Ask: feedback, intros to provider teams, and guidance for SCF positioning.

## Provider angle

We are testing a Provider Kit for paid agent-accessible APIs. The provider does not need to give agents broad credentials: the agent discovers a resource, prepares payment, and the human approves.

Good first providers:
- API credits.
- MCP tools.
- Risk/report endpoints.
- Data/search/browser services.

Ask: would you expose one resource at `0.01 USDC` testnet for a pilot?

## Feedback questions

1. What did you think this product does?
2. Where did trust increase or decrease?
3. What would make you comfortable approving the first payment?
4. Which provider or use case should we test first?
5. Who else should see this?
